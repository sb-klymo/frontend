# New API Client

Add a typed wrapper around the auto-generated OpenAPI SDK for a specific domain. Optionally wrap in TanStack Query hooks.

## Usage

`/new-api-client $ARGUMENTS`

Example: `/new-api-client bookings`

## Arguments

- Domain name (matches a backend router prefix — e.g. `bookings`, `chat`, `onboarding`)

## Steps

1. Ensure the backend has exported the latest OpenAPI schema: backend runs `python scripts/export_openapi.py`, commits `docs/schemas/openapi.json`.
2. Regenerate the SDK: `npm run gen:api` (reads schema → writes `src/lib/api/generated/`).
3. Create `src/lib/api/<domain>.ts` with typed wrappers + TanStack Query hooks.
4. Add a test in `src/lib/api/<domain>.test.ts`.

## Template — `src/lib/api/<domain>.ts`

```ts
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";
import * as sdk from "./generated/sdk.gen";
// The generated SDK exposes operation-name-based functions, e.g. `sdk.getBookings()`, `sdk.postBookings(data)`.
// Check src/lib/api/generated/types.gen.ts for the exact exports.

// =======================================================
// Raw callers — typed HTTP functions
// =======================================================

export async function list<Domain>s() {
  const { data, error } = await sdk.get<Domain>s();
  if (error) throw new Error("Failed to load");
  return data!;
}

export async function create<Domain>(payload: sdk.Create<Domain>Request) {
  const { data, error } = await sdk.post<Domain>s({ body: payload });
  if (error) throw new Error("Failed to create");
  return data!;
}

// =======================================================
// TanStack Query hooks — recommended usage
// =======================================================

export function use<Domain>s() {
  return useQuery({
    queryKey: ["<domain>s", "list"],
    queryFn: list<Domain>s,
  });
}

export function useCreate<Domain>() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: create<Domain>,
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: ["<domain>s"] });
    },
  });
}
```

## Auth injection

The generated SDK should be configured with a base URL and an auth token provider once, in `src/lib/api/client.ts`:

```ts
// src/lib/api/client.ts
import { client } from "./generated/client.gen";

import { createClient } from "@supabase/supabase-js";

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
);

client.setConfig({
  baseUrl: process.env.NEXT_PUBLIC_API_URL!,
  async auth() {
    const { data } = await supabase.auth.getSession();
    return data.session?.access_token ?? "";
  },
});
```

Import this file once at app startup (e.g. in `src/providers/Providers.tsx`).

## Test template

```ts
// src/lib/api/<domain>.test.ts
import { describe, it, expect, vi } from "vitest";

// Mock the generated SDK
vi.mock("./generated/sdk.gen", () => ({
  get<Domain>s: vi.fn(() => Promise.resolve({ data: [{ id: "1" }] })),
}));

import { list<Domain>s } from "./<domain>";

describe("<domain> API client", () => {
  it("returns the list from the SDK", async () => {
    const result = await list<Domain>s();
    expect(result).toEqual([{ id: "1" }]);
  });
});
```

## Reminders

- Never commit `src/lib/api/generated/` (it's regenerated by `npm run gen:api`)
- Always invalidate related queries on successful mutations
- Query keys follow `[entity, scope, params]` pattern
- Use generated Zod schemas (via hey-api plugin) for React Hook Form validation
- When backend changes, run `npm run gen:api` before touching frontend code that uses the new endpoint
